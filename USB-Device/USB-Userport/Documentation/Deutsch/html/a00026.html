Zentrale des USB-Userport.

Der Quellcode für den Kern des USB-Userport. Die Datei enthält die grundsätzlichen Funktionen und die Steuerlogik.

Funktion die vom LUFA HID-Klassentreiber aufgerufen wird, wenn der Host einen USB-IN-Report anfordert. Der übermittelte Report wird, abhängig von ReportType und ReportID, zusammengestellt. Die Funktion wird auch aufgerufen wenn der Host einen USB- Feature-Report anfordert. Eine spezielle Überlegung gilt den INTERRUPT-IN Reports. Die werden hier ebenfalls zusammengebaut und enthalten asynchrone Daten. In dem Fall ist die angeforderte Report-ID '0'. Nun, das ist so etwas seltsam weil der Host ganz gezielt nach dem Report fragt. Für den Bus ist es also nicht asynchron. Und weil der Versand des Reports warten muss bis der Host danach fragt, ist es auch kein Interrupt im eigentlichen Sinn. Am besten stellt man sich das als universelle Anfrage nach unspezifischen Daten vor, die das Gerät jetzt übermitteln darf - aber nicht muss. Der Host empfängt den Report und leitet die Daten an ein Programm weiter. Aus Sicht dieses Programms kommen die Daten asynchron herein und unterbrechen den normalen Programmablauf.

Zu beachten
Die Reaktion muss innerhalb einer bestimmten Zeit erfolgen. Anderenfalls nimmt der Host an, dass das Gerät fehlerhaft arbeitet.
Parameter
[in]HIDInterfaceInfozeigt auf die angewählte HID Class Treiber Schnittstellenkonfiguration und Status Struktur.
[in]ReportIDübergibt die vom Host angeforderte Report ID. '0' wenn es sich um die INTERRUPT-IN Anforderung handelt.
[in]ReportTypeist der Typ des Reports, entweder HID_REPORT_ITEM_In oder HID_REPORT_ITEM_Feature.
[in]ReportDatazeigt auf den Puffer in dem die Datenbytes abzulegen sind.
[in]ReportSizedefiniert die Reportlänge in Bytes (bzw. '0' wenn kein Report gesendet werden soll.
Rückgabe
Boolean true wenn ein Report gesendet werden soll, false wenn kein Report zurück geht. Die LUFA-Automatik ist wegen der ReportIDs nicht nutzbar.

Benutzt FEATURE_ID_REFLASH, FEATURE_SIZE_REFLASH, GPIO1_GetDirections(), GPIO1_GetLines(), GPIO2_GetDirections(), GPIO2_GetLines(), IRQmaskGPIO1, IRQmaskGPIO2, LEDs_GetAll(), previousGPIO1, previousGPIO2, REPORT_ID_DEVICE_LEDS, REPORT_ID_DIRS_GPIO1, REPORT_ID_DIRS_GPIO2, REPORT_ID_GET_ADC4, REPORT_ID_GET_ADC5, REPORT_ID_GET_ADC6, REPORT_ID_GET_ADC7, REPORT_ID_IRQ_GPIO, REPORT_ID_IRQEN_GPIO, REPORT_ID_LINES_GPIO1, REPORT_ID_LINES_GPIO2, REPORT_ID_SERVO_PWM, REPORT_SIZE_DEVICE_LEDS, REPORT_SIZE_GET_ADC, REPORT_SIZE_GET_GPIO, REPORT_SIZE_IRQ_GPIO, REPORT_SIZE_IRQEN_GPIO, REPORT_SIZE_SERVO, RXLED, RXLED_POS, TXLED und TXLED_POS.

Funktion die vom LUFA HID-Klassentreiber aufgerufen wird, wenn der Host einen USB-OUT-Report gesendet hat. Der übermittelte Report wird, abhängig von ReportType und ReportID, verarbeitet. Die Funktion wird auch aufgerufen wenn der Host einen USB- Feature-Report gesendet hat.

Parameter
[in]HIDInterfaceInfozeigt auf die angewählte HID Class Treiber Schnittstellenkonfiguration und Status Struktur.
[in]ReportIDübergibt die Report ID vom Host.
[in]ReportTypeist der Typ des Reports, entweder HID_REPORT_ITEM_Out oder HID_REPORT_ITEM_Feature.
[in]ReportDatazeigt auf den Puffer in dem die Datenbytes zu finden sind.
[in]ReportSizedefiniert die Reportlänge in Bytes.

Benutzt FEATURE_ID_REFLASH, GPIO1_ChangeDirections(), GPIO1_ChangeLines(), GPIO2_ChangeDirections(), GPIO2_ChangeLines(), IRQmaskGPIO1, IRQmaskGPIO2, LEDS_NO_LEDS, LEDs_SetAll(), REPORT_ID_DEVICE_LEDS, REPORT_ID_DIRS_GPIO1, REPORT_ID_DIRS_GPIO2, REPORT_ID_IRQEN_GPIO, REPORT_ID_LINES_GPIO1, REPORT_ID_LINES_GPIO2, REPORT_ID_SERVO_PWM, REPORT_ID_SET_ADC, RXLED, RXLED_POS, TXLED und TXLED_POS.
