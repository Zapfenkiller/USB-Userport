Ausführliche Beschreibung

Bedienung der GPIOs mit dem Sparkfun ProMicro.

API um die 18 IO-Leitungen des Sparkfun ProMicro zu bedienen. Durch diese Routinen wird der Kern des USB-Userport ein wenig unabhängiger von der unterlagerten Hardware. Man nennt das neudeutsch "Hardware Abstraction Layer" (HAL). Einige IO-Anschlüsse des ATmega32U4 sind nicht auf Platinenanschlüsse herausgeführt. Diese IO-Anschlüsse werden auf sinnvolle Voreinstellungen gebracht und von weiteren Zugriffen ausgeschlossen.

◆ GPIO1_ChangeDirections()

static void GPIO1_ChangeDirections ( const uint16_t  GPIOdirs,
const uint16_t  GPIOmask 
)
inlinestatic

stellt die Richtung der IO-Leitungen für GPIO1 ein. GPIObits verändert die jeweilige Leitungsrichtung nur, wenn die entsprechende Bitposition in GPIOmask gesetzt ist. Auf die Art können bestimmte Bits ausgewählt werden.

Parameter
[in]GPIOdirsdefiniert das Bitmuster für die Datenrichtungen. Die Leitung ist ein Ausgang wenn das Bit gesetzt ist ('1').
[in]GPIOmaskdefiniert welche Bits geschrieben werden: '1' = schreiben, '0' = nicht ändern.
Zu beachten
Von aussen nicht erreichbare IO-Leitungen des µC werden nicht angesprochen.
